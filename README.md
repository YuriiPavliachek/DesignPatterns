# DesignPatterns
# Шаблони проєктування на C#

## Мета роботи
Ознайомитися з основними шаблонами проєктування та продемонструвати їх практичне застосування на мові програмування C#.

## Виконані завдання
У цьому проєкті реалізовано по одному шаблону з кожної категорії:

- Породжувальний (Creational) — **Prototype**
- Структурний (Structural) — **Bridge**
- Поведінковий (Behavioral) — **Observer**

Для кожного шаблону наведено:
- пояснення проблеми, яку він вирішує
- опис ідеї реалізації
- UML-діаграму
- приклад реалізації на C#
- коротку демонстрацію роботи

---

# 1. Prototype (Прототип)

## Проблема
Іноді створення об'єкта є складним або дорогим процесом. Якщо потрібно створювати багато подібних об'єктів, створювати їх з нуля кожного разу може бути неефективно.

## Ідея
Патерн **Прототип** дозволяє створювати нові об'єкти шляхом копіювання вже існуючих.  
Об'єкт-прототип реалізує метод `Clone()`, який створює копію об'єкта.

## Приклад застосування
У комп'ютерних іграх, коли потрібно створити копії персонажа (наприклад, ілюзії героя).

## UML-структура


Prototype

Clone()

▲
|

ConcretePrototype

Clone()


## Реалізація

Клас `Character` реалізує інтерфейс `IPrototype` та містить метод `Clone()` для створення копії об'єкта.

# 2. Bridge (Міст)

## Проблема
Коли система має дві незалежні ієрархії класів, кількість комбінацій класів може швидко збільшуватися.

## Ідея
Патерн **Bridge** розділяє абстракцію та реалізацію, дозволяючи змінювати їх незалежно.

## Приклад
Система повідомлень:
- тип повідомлення (звичайне, термінове)
- спосіб відправки (Email, SMS)

## UML-структура


Abstraction

Send()

|

RefinedAbstraction

Implementor

SendMessage()

▲
|

ConcreteImplementor


## Реалізація
- `Message` — абстракція
- `IMessageSender` — інтерфейс реалізації
- `EmailSender`, `SmsSender` — конкретні реалізації

---

# 3. Observer (Спостерігач)

## Проблема
Потрібно повідомляти декілька об'єктів про зміну стану іншого об'єкта.

## Ідея
Патерн **Observer** дозволяє об'єктам підписуватися на зміни іншого об'єкта та отримувати сповіщення автоматично.

## Приклад
Метеостанція повідомляє різні дисплеї про зміну температури.

## UML-структура


Subject

Attach()

Detach()

Notify()

▲
|

ConcreteSubject

Observer

Update()

▲
|

ConcreteObserver


## Реалізація
- `WeatherStation` — видавець
- `IObserver` — інтерфейс спостерігача
- `PhoneDisplay`, `ComputerDisplay` — підписники

# Демонстрація роботи

При запуску програм демонструється робота кожного патерну:

- створення копії об'єкта (Prototype)
- відправка повідомлень різними способами (Bridge)
- автоматичне повідомлення спостерігачів про зміну стану (Observer)
